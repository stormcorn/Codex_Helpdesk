# Helpdesk UI/UX 設計文件（UI/UX Design Guidelines）

本文件定義目前 UI/UX 設計目標、頁面結構、互動模式與手機版檢查標準。

## 1. 設計目標
- 讓工單處理流程「快讀、快回覆、快切換」
- 用明確狀態標籤與視覺差異降低誤操作
- 支援桌機與手機操作，不水平爆版
- 維持內部系統導向的資訊密度，但避免過度擁擠

## 2. 使用者流程（核心）
## 2.1 未登入使用者
1. 開啟系統
2. 登入或註冊
3. 註冊時先選擇部門群組（第 1 步）

## 2.2 一般使用者（USER）
1. 建立工單（Helpdesk tab）
2. 查看進行中工單 / 封存工單
3. 在工單明細中回覆訊息與附件
4. 接收通知並快速跳轉工單

## 2.3 IT / ADMIN
1. 進入進行中工單
2. 變更狀態 / 回覆 / 刪除
3. （ADMIN）切到 Members tab 管理成員、群組、分類、稽核

## 3. 畫面資訊架構（IA）
## 3.1 Header 區
- 使用者資訊
- 通知入口（badge）
- 登出

## 3.2 Dashboard Tabs
- `helpdesk`：建單 + 工單列表（一般使用者主要頁）
- `itdesk`：工單處理列表
- `archive`：封存工單
- `members`：管理後台（Admin）

## 3.3 工單卡片（Ticket Card）
資訊層級（由上而下）：
1. 工單標題列（編號 + 主旨）
2. Meta 行：
   - `工號 姓名`
   - 建立時間
   - 群組 / 分類
3. 狀態/優先層級/主管確認標記
4. 展開內容：
   - 問題描述
   - 附件
   - 訊息串
   - 狀態歷程
   - 回覆區（進行中工單）

## 4. 互動規範
## 4.1 工單展開/收合
- 使用可點擊標題區 (`ticket-toggle`)
- 展開收合有轉場效果（避免突兀跳動）

## 4.2 通知跳轉
- 點通知後：
  - 切換正確頁籤
  - 展開對應工單
  - 滾動到中間位置
  - 高亮工單卡片（jump highlight）

## 4.3 即時更新（WebSocket）
- 後台收到他人操作時：
  - 自動刷新列表與通知
  - 高亮受影響工單
- UX 原則：
  - 不彈出阻斷式 modal
  - 以列表更新 + 高亮提示為主

## 4.4 回覆區（全身份）
- 顯示位置：工單展開內容底部（進行中工單）
- 內容：
  - 回覆文字輸入框
  - 送出回覆按鈕
  - 回覆附件上傳（可多檔，單檔 < 5MB）
  - 已選檔案清單

## 5. 狀態與視覺語言
## 5.1 工單狀態色彩
- `OPEN`
- `PROCEEDING`
- `PENDING`
- `CLOSED`
- `DELETED`

原則：
- 狀態標籤、卡片左側線條、篩選 chip 顏色語義一致

## 5.2 優先層級
- `GENERAL`：中性色
- `URGENT`：暖色 / 明顯警示色

## 5.3 已刪除工單
- 標題與內容視覺淡化
- 刪除線 / 刪除 badge
- 不可再執行狀態變更

## 6. 表單 UX 規範
## 6.1 註冊流程
- 多步驟呈現（stepper）
- 第 1 步先選群組，降低後續建立工單時資料不一致
- 群組欄位右側固定顯示提醒：
  - `若無可選群組，請先聯繫管理員建立。`

## 6.2 工單建立表單
- 必填欄位清楚標示
- 附件限制（單檔 < 5MB）顯示於欄位文案
- 錯誤訊息顯示在表單區域底部（非 alert）

## 6.3 管理後台表單
- 新增/指派操作採 row layout
- 在手機版需允許換行並保持按鈕可點擊

## 7. 響應式（Responsive）規範
## 7.1 目標
- 手機寬度下無水平溢出（no horizontal overflow）
- 長文字/檔名可換行
- 表格可水平捲動或改用分段呈現

## 7.2 已採用的策略
- `box-sizing: border-box`
- flex 容器允許 `wrap`
- 關鍵區塊加 `min-width: 0`
- `overflow-wrap: anywhere`
- 手機版 filter 區改單欄
- 表格區域可 `overflow-x: auto`

## 7.3 手機版驗收清單（每次 UI 改動）
- [ ] 登入/註冊頁沒有超出螢幕
- [ ] 工單篩選列沒有超出螢幕
- [ ] 工單卡片標題 / meta 可換行
- [ ] 回覆附件區沒有超出螢幕
- [ ] Members/Admin 列表可操作（表格可捲動）
- [ ] 沒有整頁水平滾動條（除表格區塊外）

## 8. UI 文案與語言規範（摘要）
- 介面主語言：繁體中文
- 狀態值可保留英文（OPEN/PROCEEDING...）以利 IT 溝通
- 錯誤訊息應具體（例：檔案超過 5MB）
- 危險操作需二次確認（刪除）

## 9. 後續優化建議（UI/UX）
1. 即時連線狀態指示（已連線 / 重連中）
2. 回覆區附件拖放上傳
3. 行動裝置下管理後台表格卡片化
4. 工單明細分區折疊（訊息/附件/歷程）
